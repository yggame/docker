服务器账户与环境使用指南

面向多人共享服务器的账户初始化、个人 Conda 环境、CUDA、数据盘、Tmux 后台任务和网络重新登录流程。命令已整理为可复制代码块。

账户初始化启用系统 Anaconda base 环境，并通过 bash 配置生效。
个人环境每位用户创建自己的 Conda 环境，避免互相影响。
数据存放代码和数据放在 /hdd，减少系统盘压力。
后台任务用 Tmux 保持训练或实验进程持续运行。

账户与基础环境

新账号创建后，主目录位于 /home/用户名。首次使用时先初始化系统默认 Anaconda 环境。

bash
/opt/anaconda3/bin/conda init bash
source ~/.bashrc
初始化后会进入系统默认的 base 环境。这个环境只作为基础环境使用，后续建议每个人创建自己的独立环境运行程序。

创建个人 Conda 环境

在 base 环境基础上创建个人环境，之后所有安装和运行都在自己的环境里完成。

bash 模板
conda create -n 环境名 python=3.12
conda activate 环境名

如果想继承 base 环境中的 Anaconda 基础包，可在创建时追加 anaconda。

bash 可选
conda create -n 环境名 python=3.12 anaconda
conda activate 环境名
新建环境默认没有 PyTorch。机器 CUDA 版本为 12.6，按项目需要安装对应框架和依赖。

安装 PyTorch

需要 PyTorch 时，可在官方历史版本页面查找匹配版本：PyTorch Previous Versions。

bash 示例
pip install torch==2.2.1 torchvision==0.17.1 torchaudio==2.2.1 --index-url https://download.pytorch.org/whl/cu121
示例命令使用的是 cu121 wheel。实际安装时请根据项目所需的 PyTorch、CUDA wheel 版本选择。

自动激活个人环境

如果希望每次打开终端都自动进入个人环境，可把激活命令放到 ~/.bashrc 最后一行。

编辑配置文件
bash
vim ~/.bashrc
在最后一行添加
bash
conda activate 环境名
退出 vim 后立即生效
bash
source ~/.bashrc

CUDA 12.6 配置

如需显式使用 CUDA 12.6，可在 ~/.bashrc 中加入 CUDA 路径。

bash
vim ~/.bashrc
bash 添加到 ~/.bashrc
export PATH=/usr/local/cuda-12.6/bin:$PATH
export LD_LIBRARY_PATH=/usr/local/cuda-12.6/lib64:$LD_LIBRARY_PATH
bash
source ~/.bashrc
这里已将原始写法中的 export export PATH 规范为正确命令：export PATH=...。

数据盘与工作目录

为节省系统盘空间，所有人的数据和代码建议放在 /hdd 下自己的文件夹中。

bash
cd /hdd
mkdir 文件夹名

如果希望每次启动终端自动进入自己的数据目录，可在 ~/.bashrc 最后一行加入：

bash 添加到 ~/.bashrc
cd /hdd/文件夹名
如果同时配置了 conda activate 环境名 和 cd /hdd/文件夹名，建议先激活环境，再切换目录。

Tmux 终端复用

建议用 Tmux 运行长时间任务。断开 SSH 或关闭当前窗口后，会话和其中的进程仍可在后台继续运行。

创建会话tmux new -s <session-name>
分离会话按 Ctrl+b，再按 d
查看会话tmux ls
重新接入tmux attach -t <session-name>
杀死会话tmux kill-session -t <session-name>
重命名会话tmux rename-session -t <old-name> <new-name>
bash 速查
tmux new -s <session-name>
tmux ls
tmux attach -t <session-name>
tmux kill-session -t <session-name>
tmux rename-session -t <old-name> <new-name>

网络重新登录

如果发现服务器与外网断连，可使用 elinks 登录网络通并重新认证。

bash
elinks wlt.ustc.edu.cn

打开后按照网络通登录流程完成认证即可。
